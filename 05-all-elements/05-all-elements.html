All elements

<a>

The anchor element (<a>) is one of the fundamental elements in HTML.

It allows a user to click on the link text it provides, and navigates to that link's location.

<abbr>

The abbreviation element (<abbr>) is one of the fundamental elements in HTML.

It allows a user to use an abbreviation(s) or acronym(s) in a text.

<>article

The article element (<article>) is one of the fundamental elements in HTML.

It allows a user to reprent a section of a page which consists of a composition that forms an independent part of a document, page oe site. For examples a forum post, a magazine, a user-submitted comment.

<>b

The b element (<b>) is one of the fundamental elements in the HTML.

It allows a user to represent a span of text to be stylistically offset from the normal prose without conveying any extra importance. Examples are key words in a document abstract, product names in a review, or other spans of text whose typical typographic presentation is bold.

<body>

The body element (<body>) is one of the fundamental elements in HTML.

It allows a user to represent the main content of the document.

<button>

The button element (<button>) is one of the fundamental elements in HTML.

It allows a user to show a button on a document, and if the button is disabled, then the user agent should allow the user to activate the button.

<caption>

The caption element (<caption>) is one of the fundamental elements in HTML.

It allows a user to represent the title of a table which is its parent, if it has a parent and thah is a table element.

<code>

The code element (<code>) is one of the fundamental elements in HTML.

It represents a fragment of computer code and amaizingly, it can be an exel element name, a filename and even a computer program.

<dd>

The description, definition element (<dd>) is one of the fundamental elements in HTML.

It stands for description, definition, or value, part of a term description group in a description list.

<del>

The del element (<del>) is one of the fundamental elements in HTML.

It represents a removal from the document. Thedel elements should not cross implied paragraph boundaries.

<dfn>

The dfn element (<dfn>) is one of the fundamental elements in HTML.

The dfn element represents the defining instance of a term. The paragraph, description list group, or section that is the nearest ancestor of the dfn element must also contain the definition(s) for the term given by the dfn element.

<div>

The division element (<div>) is one of the fundamental elements in HTML.

It doesn't really have special meaning at all, but what it does is to represent it s children and it can be used with the class and lang.

<DOCTYPE>

The DOCTYPE element (<DOCTYPE>) is one of the fundamental elements in HTML.

It is a required preamble and must consist of the following characters, like this: A string that is an ASCII case-insensitive match for the string

<dt>

The dt element (<dt>) is one of the fundamental elements in HTML.

It represents the term, or name, part of a term-description group in a description list (dl element).

<em>

The em element (<em>) is one of the fundamental elements in HTML.

It represents stress emphasis of its contents. The level of emphasis that a particular piece of content has is given by its number of ancestor em elements. The 'stress' being referred to is linguistic. If spoken, this stress would be emphasised pronunciation on a word that can change the nuance of a sentence. .

<footer>

The footer element (<footer>) is one of the fundamental elements in HTML.

It contains metadata about its enclosing section, such as who wrote it, links to related documents, copyright data, etc. Contact information for the section given in a footer should be marked up using the address element.

<form>

The form element (<form>) is one of the fundamental elements in HTML.

It gives a collection of form-associated elements, some of which can represent editable values that can be submitted to a server for processing.

<h1-h6>

The h1-h6 element (<h1-h6>) is one of the fundamental elements in HTML.

They stand for represanting collection of form-associated elements, some of which can represent editable values that can be submitted to a server for processing.

<head>

The head element (<head>) is one of the fundamental elements in HTML.

It represents a collection of metadata for the Document.

<header>

The header element (<header>) is one of the fundamental elements in HTML.

It is typically used to group a set of h1–h6 elements to mark up a page's title with its subtitle or tagline. header elements may, however, contain more than just the section's headings and subheadings — e.g., version history information or publication date..

<html>

The html element (<html>) is one of the fundamental elements in HTML.

The html element represents the root of an HTML document.

<i>

The i element (<i>) is one of the fundamental elements in HTML.

It represents a span of text in an alternate voice or mood, or otherwise offset from the normal prose, such as a taxonomic designation, a technical term, an idiomatic phrase from another language, a thought, a ship name, or some other prose whose typical typographic presentation is italicized.

<iframe>

The iframe element (<iframe>) is one of the fundamental elements in HTML.

It is used to represent a nested browsing context. For iframe elements in HTML documents, the attribute, if present, must have a value using the HTML syntax that consists of the following syntactic components, in the given order: Any number of comments and space characters. Optionally, a DOCTYPE. Any number of comments and space characters. The root element, in the form of an html element. Any number of comments and space characters.

<img>

The img element (<img>) is one of the fundamental elements in HTML.

It allows a user to put an image. The image given by the src attribute is the embedded content, and the value of the alt attribute is the img element's fallback content. The src attribute must be present, and must contain a valid non-empty URL potentially surrounded by spaces referencing a non-interactive, optionally animated, image resource that is neither paged nor scripted.

<ins>

The ins element (<ins>) is one of the fundamental elements in HTML.

The ins element represents an addition to the document. The ins elements should not cross implied paragraph boundaries.

<li>

The list element (<li>) is one of the fundamental elements in HTML.

It allows a user to make a list on a document.

<link>

The link element (<link>) is one of the fundamental elements in HTML.

It allows authors to link their document to other resources and it also allows reviewer to gain more informatio by clicking the link. The destination of the link(s) is given by the href attribute, which must be present and must contain a valid non-empty URL potentially surrounded by spaces.

<main>

The main element (<main>) is one of the fundamental elements in HTML.

The main element is an exact analogue of ARIA's role="main", and is designed to show screenreaders and assistive technologies exactly where main content begins, so it can be a target for a "skip links" keyboard command, for example. It could also be used for content syndication (Instapaper-ish things); mobile browsers could zoom in on main when encountering non-responsive websites. It should therefore be used once per page.

<meta>

The meta element (<meta>) is one of the fundamental elements in HTML.

The meta element represents various kinds of metadata that cannot be expressed using the title, base, link, style, and script elements. The meta element can represent document-level metadata with the name attribute, pragma directives with the http-equiv attribute, and the file's character encoding declaration when an HTML document is serialized to string form (e.g. for transmission over the network or for disk storage) with the charset attribute.

<nav>

The nav element (<nav>) is one of the fundamental elements in HTML.

It is used to represent navigation for a document. The nav element is a section containing links to other documents or to parts within the current document. Not all groups of links on a page need to be in a nav element — only groups of primary navigation links. In particular, it is common for footers to have a list of links to various key parts of a site, but the footer element is more appropriate in such cases.

<ol>

The ordered list element (<ol>) is one of the fundamental elements in HTML.

It allows a user to make a list of items, where the items have been intentionally ordered, such that changing the order would change the meaning of the list.

<option>

The option element (<option>) is one of the fundamental elements in HTML.

It represents an option in a select element or as part of a list of suggestions in a datalist element. In certain circumstances described in the definition of the select element, an option element can be a select element's placeholder label option. A placeholder label option does not represent an actual option, but instead represents a label for the select control.

<p>

The paragraph element (<p>) is one of the fundamental elements in HTML.

It allows a user to write a paragraph..

<script>

The script element (<script>) is one of the fundamental elements in HTML.

It allows authors to include dynamic script and data blocks in their documents. The element does not represent content for the user..

<section>

The section element (<section>) is one of the fundamental elements in HTML.

Represents a generic document or application section. In this context, a section is a thematic grouping of content, typically with a header, possibly with a footer. Examples include chapters in a book, the various tabbed pages in a tabbed dialog box, or the numbered sections of a thesis. A web site's home page could be split into sections for an introduction, news items, contact information.

<select>

The select element (<select>) is one of the fundamental elements in HTML.

The select element represents a control for selecting amongst a set of options.

<small>

The small element (<small>) is one of the fundamental elements in HTML.

Represents side comments such as small print. It is not intended to be presentational. The small element should not be used for extended spans of text such as multiple paragraphs, lists, or sections of text. It is only intended for short runs of text.

<source>

The source element (<source>) is one of the fundamental elements in HTML.

The source element allows authors to specify multiple alternative media resources for media elements. It does not represent anything on its own. The src attribute gives the address of the media resource. The value must be a valid non-empty URL potentially surrounded by spaces. This attribute must be present.

<span>

The span element (<span>) is one of the fundamental elements in HTML.

It doesn't mean anything on its own, but can be useful when used together with the global attributes, e.g. class, lang, or dir. It represents its children.

<style>

The style element (<style>) is one of the fundamental elements in HTML.

It allows authors to embed style information in their documents. The style element is one of several inputs to the styling processing model. The element does not represent content for the user.

<sammary>

The sammary element (<sammary>) is one of the fundamental elements in HTML.

It epresents a summary, caption, or legend for the rest of the contents of the summary element's parent details element, if any.

<table>

The table element (<table>) is one of the fundamental elements in HTML.

It represents data with more than one dimension, in the form of a table. Tables must not be used as layout aids.

<tbody>

The tbody element (<tbody>) is one of the fundamental elements in HTML.

The tbody element represents a block of rows that consist of a body of data for the parent table element, if the tbody element has a parent and it is a table.

<td>

The td element (<td>) is one of the fundamental elements in HTML.

The td element represents a data cell in a table.

<tfoot>

The tfoot element (<tfoot>) is one of the fundamental elements in HTML.

It represents the block of rows that consist of the column summaries (footers) for the parent table element, if the tfoot element has a parent and it is a table.

<th>

The th element (<th>) is one of the fundamental elements in HTML.

The th element represents a header cell in a table.

<thead>

The thead element (<thead>) is one of the fundamental elements in HTML.

It represents the block of rows that consist of the column labels (headers) for the parent table element, if the thead element has a parent and it is a table.

<title>

The title element (<title>) is one of the fundamental elements in HTML.

The title element represents the document's title or name. Authors should use titles that identify their documents even when they are used out of context, for example in a user's history or bookmarks, or in search results. The document's title is often different from its first heading, since the first heading does not have to stand alone when taken out of context.

<tr>

The tr element (<tr>) is one of the fundamental elements in HTML.

The tr element represents a row of cells in a table.

<ul>

The ul element (<ul>) is one of the fundamental elements in HTML.

The ul element represents a list of items, where the order of the items is not important — that is, where changing the order would not materially change the meaning of the list.
